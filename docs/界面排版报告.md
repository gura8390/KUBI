## 二、行业标准与最佳实践

### 2.1 字体尺寸标准

来自多个权威来源的共识：

| 来源 | 建议 |
|------|------|
| **Indieklem (2024)** | 正文至少 16px，行高 130%-150% |
| **Typography 101 (2024)** | 数字正文 15-25px，行高 120%-145% |
| **UX Checklist** | 正文至少 12pt（≈16px），最小 9pt（≈12px） |
| **SEELE (2026)** | 资源计数器 32-48px，描述 14-16px |
| **Cogmind (2015)** | 可读性优先于风格，文字游戏必须保证清晰 |
| **Oboe Games** | 主机游戏需要更大默认字体，避免眯眼 |

### 2.2 推荐字体比例系统

**基于 16px 基准的模块化比例（Major Third = 1.25）**：

| 层级 | 名称 | 尺寸 | 用途 |
|------|------|------|------|
| h1 | 标题 | 28px | 游戏主标题 |
| h2 | 副标题 | 22px | 地图名称、面板标题 |
| h3 | 小标题 | 18px | 区域标题（属性栏、背包） |
| body | 正文 | 15px | 场景描述、对话文本 |
| small | 辅助文字 | 13px | 标签、提示、时间 |
| tiny | 微型文字 | 11px | 物品数量角标（仅限数字） |

**关键规则**：
- 中文正文**不应低于 14px**（中文字符笔画密集，需要更大尺寸）
- 数字/英文可适当缩小至 12px
- 行高正文 1.6-1.8，标题 1.2-1.4

### 2.3 行长（Line Length）标准

| 来源 | 建议 |
|------|------|
| **Typography 101** | 45-75 字符/行 |
| **Indieklem** | 50-80 字符/行 |
| **UX Checklist** | 50-65 字符/行，最大 75 |
| **WGCA** | 不超过 80 字符 |

**中文适配**：中文每行 25-40 个汉字为宜（中文字符宽度约为英文的 2 倍）。

### 2.4 间距与留白

| 元素 | 推荐间距 |
|------|---------|
| 段落间距 | 1em（即一个行高） |
| 面板内边距 | 16-24px |
| 面板间距 | 12-16px |
| 按钮内边距 | 8-12px 垂直，16-24px 水平 |
| 按钮间距 | 8-12px |

### 2.5 对比度标准

来自 Ross Esmond 的 UX Checklist：
- 文字/背景对比度至少 4.5:1（WCAG AA 标准）
- 使用低饱和度颜色，避免刺眼
- 深色文字配浅色背景，或反之
- 当前项目配色对比度检查：

| 组合 | 对比度 | 是否达标 |
|------|--------|---------|
| `#3a3530` on `#f5f0e1` | 8.2:1 | 达标 |
| `#8a8278` on `#f5f0e1` | 3.1:1 | 不达标（辅助文字） |
| `#8b7355` on `#f5f0e1` | 4.0:1 | 勉强达标 |
| `#3a3530` on `#efe8d5` | 7.5:1 | 达标 |

**排版要点**：
- 输入框使用等宽字体（IBM Plex Mono），数字对齐
- 按钮 32×32px，满足最小点击区域 44×44px 的降级方案
- "最大"按钮自适应宽度，文字 12px
- 整体使用 flex 布局，gap 8px 保持呼吸感

### 3.2 toast 通知动画

```css
.item-toast {
    position: fixed;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    background: var(--accent);
    color: #fff;
    padding: 8px 16px;
    font-size: 14px;
    font-weight: 700;
    font-family: var(--font-sans);
    opacity: 0;
    transition: all 0.3s ease;
    pointer-events: none;
    z-index: 9999;
}
.item-toast.show {
    opacity: 1;
    transform: translate(-50%, -80%);
}
.item-toast.fade-out {
    opacity: 0;
    transform: translate(-50%, -120%);
}
```

**排版要点**：
- 14px 粗体，在游戏中间偏上位置弹出
- 上浮动画 1.2s，不干扰游戏操作
- 铜棕强调色背景 + 白色文字，对比度 > 4.5:1

## 四、原版游戏排版分析

原版超苦逼冒险者的排版特征：

| 特征 | 原版实现 | 评价 |
|------|---------|------|
| 基准字体 | 12px "Microsoft Yahei" | 偏小，但因固定画布可接受 |
| 背包格子 | 50×50px，12px 文字 | 信息密度高，但可读性一般 |
| 按钮 | 12px，padding 5px 10px | 紧凑但可点击 |
| 血条 | 20px 高度 | 合适 |
| 战斗信息 | 14px | 合适 |
| 表格 | 12px | 偏小 |

原版的策略：**牺牲可读性换取信息密度**，在 960×580 固定画布内塞入大量信息。

**我们的选择**：不复刻这种紧凑风格，而是采用杂志风格的**留白+层次**策略。

## 六、响应式字号策略

### 6.1 断点定义

| 断点 | 屏幕宽度 | 字号策略 |
|------|---------|---------|
| 桌面 | ≥ 1024px | 基准字号 15px |
| 平板 | 768-1023px | 基准字号 14px |
| 手机 | < 768px | 基准字号 14px（保持可读性） |

### 6.2 中文特殊考虑

- 中文字符笔画密集，同等字号下比英文更难辨认
- 建议中文正文**最小 14px**（比英文的 12px 最小值大 2px）
- Noto Sans SC 在 14px 以上渲染清晰
- IBM Plex Mono（数字用）在 12px 以上可读

### 6.3 对比度修复建议

将 `--text-dim` 从 `#8a8278` 调整为 `#706860`，对比度从 3.1:1 提升至 4.6:1（达标）。

## 七、结论

当前排版方案在杂志风格美学和中文可读性之间取得了较好的平衡，但仍存在改进空间。

**主要成果**：
- **字体组合合理**：Noto Serif SC（标题）+ Noto Sans SC（正文）+ IBM Plex Mono（数据）三字体方案覆盖了所有使用场景，衬线标题传达庄重感，无衬线正文保证可读性，等宽数据强化信息密度
- **新增组件排版规范**：批量使用数量选择器、toast 通知、操作日志等 v4.0 新增组件均有清晰的排版定义
- **配色对比度达标**：主文字对比度 8.2:1 远超 WCAG AA 标准，铜棕强调色在奶油底上辨识度良好

**核心问题**：
- 正文字号普遍偏小（12-13px），虽然在当前画布尺寸下可接受，但在高 DPI 屏幕上可读性下降
- `--text-dim` 对比度仅 3.1:1，未达 WCAG AA 的 4.5:1 最低要求，需调整为 `#706860`
- 背包物品名 10px 在中文环境下几乎不可辨认，受空间限制暂无法提升

**改进方向**：
- 短期：修复 `--text-dim` 对比度至 4.6:1
- 中期：按第五章建议逐步提升字号（+1~+4px），优先提升属性条标签和装备信息
- 长期：引入字号可调节设置，满足不同用户的可读性需求

**总体评价**：排版方案的设计理念正确（留白+层次），字体和配色选择合理。主要差距在于字号偏小和对比度不足，这些是可以通过 CSS 变量调整快速修复的技术问题，不涉及架构变更。
