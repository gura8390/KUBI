# 《超苦逼冒险者》1:1 复刻开发文档 v4.0

## 一、项目概述

### 1.1 游戏简介
《超苦逼冒险者》（Very Kubition Adventure）是一款基于HTML5的文字冒险休闲沙盒游戏。游戏以生存为核心，玩家需要在荒野中采集资源、建造设施、制作物品、与NPC交易、探索地牢，最终生存下去并变得更强大。游戏类似文字版《饥荒》，玩法新颖，富有魅力。

### 1.2 游戏类型
- 文字冒险
- 休闲沙盒
- 生存模拟
- RPG角色扮演

### 1.3 技术平台
- **前端技术**：HTML5 + CSS3 + JavaScript（原生，无框架）
- **运行环境**：现代浏览器（Chrome、Firefox、Safari、Edge）
- **无需服务器**：纯前端实现，数据本地存储（LocalStorage）

### 2.2 时间与季节系统

#### 2.2.1 时间系统
- 游戏内时间：以"小时"为单位
- 一天 = 24小时
- **行动推进**：时间仅在玩家行动时推进，不自动流逝
- 睡眠可快速推进时间并恢复状态
- 白天：6:00-18:00（资源可采集）
- 夜晚：18:00-6:00（部分资源不可采集，有鬼出没风险）
- **跨天事件**：自动收集建筑被动资源（水井/田地/陷阱）

#### 2.2.2 季节系统
| 季节 | 特殊效果 | 持续时间 | 温度影响 |
|------|---------|---------|---------|
| 春季 | 资源正常刷新，适合开局 | 30天 | 适中 |
| 夏季 | 资源正常刷新，需降温 | 30天 | 高温，需冰露/降温药剂 |
| 秋季 | 资源正常刷新，准备过冬 | 30天 | 适中 |
| 冬季 | **所有资源停产**，包括田地、水井 | 30天 | 低温，需热汤/火坑取暖 |

**冬季特殊规则：**
- 田地不产出小麦、浆果、生菜
- 水井不产出水分
- 只能通过交易、拾荒、狩猎、陷阱获取资源
- 睡觉消耗木头（1小时=1木头）
- 需要热汤、火坑等保暖措施
- **火坑效果**：有火坑时体温目标为 37°C，不会过低

#### 2.3.2 地图交互类型
- **采集**：消耗时间获取资源（3体力）
- **狩猎**：消耗时间获取肉类，有战斗风险
- **拾荒**：消耗时间获取随机资源（不可再生，3体力）
- **采水**：消耗时间获取水源（独立于采集，仅限有水源的地图）
- **探索**：发现新地点或NPC
- **交易**：与NPC交换物品

### 2.7 战斗系统

#### 2.7.1 战斗机制
- **回合制战斗**：玩家与怪物轮流攻击
- **伤害计算**：攻击力 - 防御力 = 实际伤害
- **命中判定**：根据命中率决定是否命中
- **战斗消耗**：消耗体力值
- **魔法伤害**：法杖类武器额外魔法伤害
- **逃跑机制**：成功逃跑后关闭战斗模态框，返回主界面

#### 2.7.2 魂点分配
| 属性 | 消耗 | 效果 |
|------|------|------|
| 防御+1 | 10魂 | 减少受到的伤害 |
| 命中+1 | 20魂 | 提高攻击命中率 |
| 攻击+1 | 30魂 | 提高攻击力 |

**推荐加点顺序**：防御优先，60防御可通关50层，命中10-30即可

## 三、游戏流程设计

### 3.1 开局阶段（第1-5天）

**目标**：建立基础生存设施

1. **初始资源**：木棍、面包×2、水×3、木头×5
2. **第一天行动**：
   - 前往幽暗森林
   - 采集浆果（目标12个）
   - 砍树（消耗至3点体力，约17次）
   - 采水（独立行动，成功率80%，1-2个）
   - 狩猎获取生肉（至少4个）
   - 采集草药、花、种子
3. **回家建造**：
   - 床（木头×5）→ 解锁睡觉
   - 箱子（木头×3）→ 背包 +5
   - 工桌（木头×5）→ 解锁制造
   - 炊具（石头×2+木头×2）→ 解锁烹饪
4. **制作食物**：
   - 汉堡（面包+生肉，40满）
   - 肉汤（生肉+水，15满8水）
   - 果汁（水+浆果，12水-5温）

### 3.2 探索阶段（第6-15天）

**目标**：解锁新地图，获取更多资源

1. **前往小镇**（给NPC 8个浆果解锁）：
   - 与农场主对话，用12小麦换30种子
   - 与矿队对话，开启矿洞
   - 与商人对话，获取2张券
   - 与镇长对话，开启贼窝
2. **继续采集**：铁块、零件、木头
3. **建造更多设施**：炼金桌、科研桌、水井、田地、陷阱
4. **制作纸张**：水+树皮，用于研究
5. **前往溪流**（5张纸解锁）：
   - 采水（独立行动，成功率90%，2-5个）
   - 采集树皮、生鱼
   - 与长者对话，用12纸解锁4个新地图

### 3.3 发展阶段（第16-30天）

**目标**：建立稳定资源循环

1. **解锁更多地图**：用纸张与长者交换
2. **建造田地**：每天自动获得小麦
3. **建造陷阱**：每天自动获得生肉
4. **制作药剂**：生命药剂、体力药剂、精神药剂
5. **制作装备**：骨棒、齿轮战锤
6. **准备过冬**：囤积食物、水、木头
7. **研究科技**：科研台解锁更多功能

### 3.4 冬季生存（第31-60天）

**目标**：熬过资源匮乏期

1. **资源管理**：严格控制消耗
2. **交易为主**：与商人交换所需资源
3. **陷阱使用**：自动获取猎物（冬季仍产出）
4. **热汤保暖**：木头+水=热汤（+10温）
5. **火坑取暖**：建造火坑维持体温（目标37°C）
6. **避免外出**：减少不必要的探索
7. **批量使用**：利用批量使用功能高效消耗库存食物

### 3.5 后期阶段（第61天+）

**目标**：挑战地牢，提升实力

1. **制作高级装备**：齿轮战锤、龙甲
2. **阵营选择**：食人族（吸血20%）或法师（魔法加成）
3. **挑战地牢**：逐层推进，获取魂
4. **加点提升**：防御优先，命中次之
5. **最终挑战**：魔王、星尘系列

```javascript
// 建造时立即应用效果
function applyBuildingEffect(buildingId) {
  const building = BUILDINGS[buildingId];
  if (!building) return;
  if (building.effect && building.effect.maxInventory) {
    player.maxInventory += building.effect.maxInventory;
  }
}

// 读档时重算所有效果
function applyAllBuildingEffects() {
  player.maxInventory = 10; // 重置为基础值
  player.buildings.forEach(buildingId => {
    const building = BUILDINGS[buildingId];
    if (!building || !building.effect) return;
    if (building.effect.maxInventory) {
      player.maxInventory += building.effect.maxInventory;
    }
  });
}

// 跨天时收集被动资源
function collectPassiveResources() {
  player.buildings.forEach(buildingId => {
    const building = BUILDINGS[buildingId];
    if (!building || !building.passive) return;
    Object.entries(building.passive).forEach(([itemId, amount]) => {
      addItemToInventory(itemId, amount);
    });
  });
}
```

#### 5.3.2 批量使用系统（utils.js）
```javascript
function useItemBatch(index) {
  const input = document.getElementById('batch-qty');
  let qty = input ? (parseInt(input.value) || 1) : 1;
  const slot = player.inventory[index];
  const item = ITEMS[slot.id];
  const count = slot.count || 1;
  qty = Math.max(1, Math.min(count, qty));

  // 应用效果（按数量叠加）
  if (item.effect) {
    Object.entries(item.effect).forEach(([stat, value]) => {
      if (player[stat] !== undefined) {
        player[stat] = Math.min(100, player[stat] + value * qty);
      }
    });
  }

  // 扣除物品
  slot.count = count - qty;
  if (slot.count <= 0) player.inventory.splice(index, 1);

  showMessage(`使用了${item.name}×${qty}`, 'success');
  hideModal();
  updateUI();
}
```

```javascript
collectWater: function() {
  const map = MAPS[this.currentMap];
  if (!map || !map.waterSources) {
    showMessage('这里没有水源', 'warning');
    return;
  }
  const source = map.waterSources;
  if (Math.random() < source.chance) {
    const amount = randomInt(source.min, source.max);
    if (addItemToInventory('water', amount)) {
      showMessage(`取到了水×${amount}`, 'success');
    }
  } else {
    showMessage('没有取到水', 'warning');
  }
  this.checkUnlocks();
}
```

### 9.5 关键架构决策

| 决策 | 原因 |
|------|------|
| 时间系统采用行动推进 | 避免玩家AFK时状态持续消耗 |
| 建筑功能按钮条件显示 | 避免UI混乱，符合游戏逻辑 |
| 地图解锁自动检查 | 玩家获取物品后自动触发，无需手动操作 |
| 存档直接设置currentMap | 避免读档时触发不必要的游戏逻辑 |
| 建筑效果分即时/被动 | 即时效果建造后生效，被动效果跨天收集 |
| 采水独立于采集 | 水源是关键生存资源，需要独立操作节奏 |
| toast 通知统一由调用方控制 | 避免 addItemToInventory 内部和外部重复弹出 |
