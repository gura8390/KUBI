## v8：世界演化版

> 让世界"活"起来，玩家的行为留下持续影响。

### 核心目标

- 世界状态系统完整运作
- 动态事件真正影响游戏
- 阵营关系有实际意义
- NPC 有记忆和反应
- 地图随时间变化

### 实现清单

#### 8.1 世界状态系统完善

**现状**: `engine/worldstate.js` 已实现基础框架（NPC关系、Boss击杀、阵营声望、历史记录），但未与游戏逻辑深度联动。

**需要实现**:
- [x] NPC 关系值影响交易价格（关系高则打折，关系低则涨价或拒绝交易）
- [x] Boss 击杀后地图发生变化（如击杀蜘蛛女王后蜘蛛洞穴怪物减少50%）
- [x] 阵营声望影响 NPC 对话内容和可用交易
- [x] 城镇发展等级影响商人数量和商品种类
- [x] 世界状态持久化到存档

**实现方式**:
```
npc.js: 在 showTrades() 中检查 WorldState.getNPCRelation(npcId)
map-data.js: 为地图添加 condition 字段，如 { condition: { type: 'bossKilled', target: 'spiderQueen' } }
npc.js: 在 dialogue 中根据 factionRep 选择不同台词
craft.js: 在 showBuildMenu() 中检查 townLevel
storage.js: 在 save/load 中序列化 WorldState
```

#### 8.2 动态事件系统增强

**现状**: `data/events.js` 有 12 个静态事件，`data/scenes.js` 有动态场景描述。

**需要实现**:
- [ ] 事件链：一个事件触发后解锁后续事件（如"发现废墟"→"找到日记"→"发现密室"）
- [ ] 事件后果：选择帮助 NPC 后，下次遇到该 NPC 有不同对话
- [x] 季节性事件：春天花祭、夏天收获节、秋天感恩节、冬天新年
- [x] 天气联动事件：暴风雨中可能发现沉船、雷击引发森林火灾
- [x] 玩家行为触发事件：连续战斗10次触发"战斗疲劳"、连续制作10次触发"灵感迸发"

**实现方式**:
```
data/events.js: 添加 chain 字段指向后续事件
data/events.js: 添加 condition 字段使用 ConditionSystem
data/events.js: 添加 triggerCount 字段记录触发次数
engine/eventbus.js: 监听 weather:changed 和 dayChanged 事件
```

#### 8.3 阵营关系深化

**现状**: 玩家选择食人族或法师阵营后，只有一个军需官 NPC，无后续内容。

**需要实现**:
- [ ] 阵营专属任务链（每个阵营 5 个任务）
- [x] 阵营声望等级（敌对→中立→友好→尊敬→崇拜）
- [x] 阵营专属装备和配方
- [ ] 阵营间冲突事件（遇到敌对阵营 NPC）
- [ ] 叛变机制（付出巨大代价切换阵营）

**实现方式**:
```
data/npcs.js: 添加阵营 NPC 数据
data/recipes.js: 添加阵营专属配方
js/quests.js: 添加阵营任务链
js/npc.js: 在对话中检查 factionRep 决定台词
```

#### 8.4 NPC 互动增强

**现状**: 18 个 NPC，每个有固定台词和交易，无记忆。

**需要实现**:
- [ ] NPC 记住玩家行为（帮助/欺骗/交易次数）
- [x] NPC 好感度影响对话和交易
- [ ] NPC 日夜节律（白天在特定位置，晚上回家）
- [ ] NPC 之间有关系（如矿工和商人是朋友）
- [ ] NPC 可被招募为同伴

**实现方式**:
```
data/npcs.js: 添加 schedule 字段定义日夜位置
js/npc.js: 在 interact() 中检查 WorldState.getNPCDialogueCount()
js/npc.js: 添加 companion 系统
engine/worldstate.js: 已有 recordNPCDialogue() 和 getNPCDialogueCount()
```

#### 8.5 地图变化系统

**现状**: 地图是静态的，不会随时间或玩家行为变化。

**需要实现**:
- [ ] Boss 击杀后地图变化（如击杀魔王后地牢入口关闭）
- [ ] 季节影响地图资源（春天花开、秋天果实、冬天雪景）
- [ ] 玩家行为影响地图（如在某地图建造哨塔后怪物减少）
- [ ] 隐藏地图解锁（满足特定条件后出现新区域）
- [x] 地图探索度（访问次数影响资源获取率）

**实现方式**:
```
data/maps.js: 添加 variations 字段定义不同状态
js/map.js: 在 updateMapDisplay() 中检查 WorldState
js/map.js: 在 collectResources() 中检查探索度
engine/worldstate.js: 已有 setMapChange() 和 getMapChange()
```

## v9：剧情叙事版

> 让世界有故事，玩家有目标，选择有重量。

### 核心目标

- 碎片化叙事系统完整
- 图鉴系统有深度
- 世界观清晰
- NPC 有个性
- 多结局系统

### 实现清单

#### 9.1 碎片化叙事系统

**现状**: `data/lore.js` 已有日记/遗书/石板数据，但未集成到游戏中。

**需要实现**:
- [ ] 收集品系统：在地图探索时随机发现日记碎片
- [ ] 叙事解锁：收集特定碎片组合解锁隐藏内容
- [ ] 世界观构建：通过碎片逐步揭示世界历史
- [ ] 叙事影响游戏：某些碎片提供永久加成或解锁配方
- [ ] 叙事进度追踪：图鉴中显示已收集/未收集

**实现方式**:
```
data/lore.js: 重新创建，定义收集品数据
js/map.js: 在 explore() 中添加收集品掉落逻辑
js/encyclopedia.js: 添加收集品图鉴
data/events.js: 添加叙事触发事件
```

#### 9.2 图鉴系统完善

**现状**: `js/encyclopedia.js` 已有基础框架，但内容不完整。

**需要实现**:
- [x] 怪物图鉴：击杀数、掉落物、弱点、背景故事
- [x] 物品图鉴：制作次数、用途、来源、描述
- [ ] 地图图鉴：访问次数、资源分布、怪物列表
- [ ] NPC 图鉴：对话次数、关系值、交易记录
- [ ] 收集品图鉴：已收集碎片、解锁进度

**实现方式**:
```
js/encyclopedia.js: 扩展 showEncyclopedia() 方法
data/scenes.js: 添加更多场景描述
js/battle.js: 在 victory() 中记录怪物数据
js/craft.js: 在制作成功时记录物品数据
```

#### 9.3 世界观构建

**现状**: 游戏有17张地图和48种怪物，但缺乏统一的世界观。

**需要实现**:
- [ ] 世界历史：通过收集品和NPC对话揭示
- [ ] 地图关联：地图之间有地理和故事联系
- [x] 怪物起源：每种怪物有背景故事
- [ ] 阵营历史：食人族和法师的起源故事
- [ ] 世界规则：魔法系统、元素系统、灵魂系统的解释

**实现方式**:
```
data/lore.js: 定义世界观数据
data/scenes.js: 添加地图背景描述
data/monsters.js: 添加怪物背景描述
js/npc.js: 在对话中添加世界观信息
```

#### 9.4 NPC 个性深化

**现状**: 5个NPC有个性化台词，其余NPC台词通用。

**需要实现**:
- [ ] 所有NPC有独特性格（通过台词体现）
- [ ] NPC对玩家行为有反应（帮助/欺骗/忽视）
- [ ] NPC之间有关系网
- [ ] NPC有个人故事线
- [ ] NPC对话随游戏进度变化

**实现方式**:
```
data/npcs.js: 为每个NPC添加personality字段
js/npc.js: 根据personality选择台词
data/npcs.js: 添加对话树数据
js/npc.js: 实现对话树逻辑
```

#### 9.5 多结局系统

**现状**: 游戏只有一个结局（击败灭世者）。

**需要实现**:
- [ ] 好结局：击败灭世者，世界恢复和平
- [ ] 坏结局：被灭世者击败，世界毁灭
- [ ] 隐藏结局：通过特定条件触发（如收集所有碎片、完成所有任务）
- [ ] 阵营结局：选择不同阵营有不同的结局动画
- [ ] 结局影响：不同结局解锁不同新游戏+内容

**实现方式**:
```
js/game.js: 添加结局判定逻辑
data/events.js: 添加结局触发事件
js/encyclopedia.js: 添加结局图鉴
```

## v10：玩法深化版

> 让游戏有深度，玩家有选择，挑战有层次。

### 核心目标

- 职业系统
- 装备流派
- Boss 机制
- 多周目
- 高难度挑战

### 实现清单

#### 10.1 职业系统

**现状**: 游戏没有职业概念，所有玩家属性相同。

**需要实现**:
- [ ] 3个基础职业：战士（高攻击/防御）、法师（高魔法/智力）、游侠（高敏捷/命中）
- [ ] 职业专属技能（每个职业4个技能）
- [ ] 职业专属装备
- [ ] 转职系统（达到一定等级后可转职为高级职业）
- [ ] 职业影响属性成长

**实现方式**:
```
data/skills.js: 添加职业专属技能
js/player.js: 添加职业字段和属性成长公式
js/battle.js: 根据职业使用不同技能
data/items.js: 添加职业专属装备
```

#### 10.2 装备流派

**现状**: 装备只有攻击/防御数值，无流派区分。

**需要实现**:
- [ ] 暴击流：高暴击率、暴击伤害
- [ ] 吸血流：攻击回复生命
- [ ] 反击流：受击后反击
- [ ] 魔法流：魔法伤害、魔法抗性
- [ ] 坦克流：高防御、高生命、嘲讽

**实现方式**:
```
data/items.js: 为装备添加特殊属性
js/battle.js: 在攻击/受击时检查装备特效
data/buffs.js: 添加流派相关Buff
```

#### 10.3 Boss 机制

**现状**: Boss只是血量更高的普通怪物，无特殊机制。

**需要实现**:
- [x] 阶段转换：Boss血量低于50%/25%时改变行为
- [ ] 召唤小怪：Boss定期召唤小怪
- [ ] AOE攻击：Boss对玩家造成范围伤害
- [x] 减益技能：Boss对玩家施加Debuff
- [ ] 弱点系统：Boss有特定弱点，使用对应元素造成额外伤害

**实现方式**:
```
data/monsters.js: 为Boss添加 phase 和 ability 字段
engine/ai.js: 实现Boss型AI的阶段转换逻辑
js/battle.js: 在怪物回合检查Boss技能
```

#### 10.4 多周目系统

**现状**: 游戏通关后无后续内容。

**需要实现**:
- [ ] 新游戏+：保留部分进度（等级、装备、配方）
- [ ] 难度递增：每周目怪物属性提升
- [ ] 周目专属内容：每周目解锁新地图/怪物/NPC
- [ ] 周目成就：完成特定周目解锁成就
- [ ] 周目排行榜：记录最快通关时间

**实现方式**
```
js/storage.js: 添加周目数据到存档
js/game.js: 在通关后触发新游戏+
data/monsters.js: 根据周目调整怪物属性
js/battle.js: 根据周目调整战斗难度
```

#### 10.5 高难度挑战

**现状**: 游戏难度固定，无挑战模式。

**需要实现**:
- [ ] 无尽模式：地牢无限层，每层难度递增
- [ ] 挑战模式：特定条件下完成任务（如不用药剂通关）
- [ ] 限时挑战：在规定时间内完成目标
- [ ] Boss Rush：连续挑战所有Boss
- [ ] 随机挑战：随机生成挑战条件

**实现方式**
```
js/map.js: 添加无尽模式逻辑
js/game.js: 添加挑战模式入口
data/events.js: 添加挑战触发事件
js/battle.js: 添加Boss Rush逻辑
```

## 实现原则

1. **数据驱动**: 所有新内容都在 data/ 中定义，逻辑层只读取和执行
2. **引擎通用**: 引擎层不包含游戏专有内容
3. **渐进式**: 每个版本独立可玩，不依赖后续版本
4. **向后兼容**: 新版本不破坏旧存档
5. **测试优先**: 每个新功能都要有对应的测试验证
